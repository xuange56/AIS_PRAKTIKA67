# Концепция проекта «kursa4» — веб-система управления кинотекой

## 1. Цели и ценность
- Предоставить пользователям удобный каталог фильмов с поиском, фильтрацией и карточками фильмов.
- Дать возможность оставлять отзывы и оценки, повышая вовлечённость.
- Обеспечить администраторам инструменты управления контентом (фильмы, жанры, актёры, режиссёры) и пользователями.
- Автоматизировать получение постеров и базовых метаданных через TMDB API, уменьшая ручной ввод.

## 2. Основные сценарии
- **Пользователь (гость/авторизованный):**
  - Просмотр каталога и страницы фильма.
  - Поиск/фильтрация по названию, жанру, году.
  - Чтение отзывов; добавление отзывов и оценок (авторизованные).
- **Администратор:**
  - CRUD для фильмов, жанров, актёров, режиссёров.
  - Модерация отзывов и управление пользователями/ролями.
  - Массовое обновление постеров через TMDB.

## 3. Ключевые требования
- Аутентификация и авторизация (роль User/Admin).
- Каталог фильмов с пагинацией, сортировкой, фильтрами.
- Отзывы: оценка 1–10, текст, дата публикации.
- Интеграция с TMDB для постеров (кеширование запросов).
- Логирование (Serilog), валидация (FluentValidation/DataAnnotations).
- Поддержка MySQL через EF Core; миграции для развёртывания.

## 4. Архитектура и технологии
- **Стек:** .NET 8, ASP.NET Core MVC, EF Core + MySQL (Pomelo), Razor Views, Bootstrap.
- **Слои:**
  - Controllers (Web/MVC)
  - Services (например, `TMDBService` для интеграции с TMDB)
  - Data (EF DbContext, миграции)
  - Models (сущности домена)
  - Views (Razor UI)
  - Tests (MSTest + Moq) — `kursa4.Tests/UnitTests`
- **Кеширование:** IMemoryCache для ответов TMDB.
- **Логирование:** Serilog + sinks (консоль/файл при необходимости).

## 5. Данные и доменная модель
- **Movie**: Title, OriginalTitle, Description, ReleaseYear, Duration, Rating, PosterURL, GenreID, DirectorID, связи с Roles и Reviews.
- **Review**: MovieID, UserID, Rating (1–10), Text, PublicationDate, CreatedAt/UpdatedAt.
- **User**: FirstName, LastName, Email, Password (хеш), Rank (роль), RegistrationDate.
- **Genre, Director, Actor, Country, Role (связь Movie–Actor с персонажем)**.

## 6. Интеграции
- **TMDB API**: поиск фильма и постера; кеширование результатов; graceful degradation при ошибках API.
- Возможное будущее: отправка email-уведомлений (SMTP), соц. логин — позже.

## 7. Нефункциональные требования
- Производительность: поисковые запросы до ~1 сек, главная <2 сек; кеш для TMDB.
- Безопасность: хеширование паролей, валидация ввода, защита от XSS/CSRF, ролевой доступ.
- Масштабируемость: отделение слоёв, возможность выноса статики и БД на отдельные сервисы.
- Надёжность: обработка ошибок внешних API, логирование.

## 8. Тестирование
- **Unit-тесты (MSTest + Moq)**: модели (инициализация, валидация), `TMDBService` (валидация входных данных, успешные/ошибочные ответы API, кеширование).
- **Интеграционные тесты** (потенциал): контроллеры MVC с тестовым InMemory DB.
- **Регресс**: `dotnet test kursa4.Tests/kursa4.Tests.csproj`.

## 9. Развёртывание и окружения
- **Dev:** локально, SQLite/MySQL, переменные окружения для секретов.
- **Prod:** MySQL, миграции EF (`dotnet ef database update`), конфиг через `appsettings.*` и переменные среды, ротация логов.
- **CI/CD (перспектива):** GitHub Actions/Azure DevOps — сборка, тесты, деплой.

## 10. План развития (roadmap)
- MVP: каталог, карточка фильма, отзывы, базовая админка, постеры TMDB.
- Улучшения: расширенный поиск (жанр/год/рейтинг), сортировка, пагинация.
- Дополнительно: рекомендации, закладки/избранное, модерация отзывов, экспорт/импорт данных.
- UI/UX: улучшить адаптивность, предпросмотр постеров, ленивая загрузка изображений.

## 11. Риски и меры
- Недоступность TMDB → кеш + фолбэк на локальные постеры.
- Некачественные данные пользователей → валидация + модерация.
- Рост нагрузки на БД → индексы, кеширование, возможный read-replica.

## 12. Навигация по репозиторию
- `kursa4/` — веб-приложение (Controllers, Models, Views, Data, Services).
- `kursa4.Tests/UnitTests/` — все модульные тесты.
- `InitialCreateScript.sql` — SQL-скрипт структуры.
- `README.md` — подробное описание курсовой работы.
