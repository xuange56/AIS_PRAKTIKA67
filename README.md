# Введение

Киноиндустрия в настоящее время переживает этап активной цифровой трансформации, вызванной стремительным ростом объемов производимого контента и изменением потребительских привычек. В условиях жесткой конкуренции между стриминговыми платформами и традиционными медиасервисами операторам необходимо внедрять инновационные решения для эффективного управления каталогами фильмов, обеспечения качественного пользовательского опыта и оптимизации процессов контент-менеджмента. Ключевую роль в этом играют специализированные информационные системы управления кинотекой, способные автоматизировать весь цикл работы с медиаконтентом – от каталогизации и описания фильмов до анализа пользовательских предпочтений и формирования рекомендаций.

Актуальность данного исследования определяется стремительной цифровизацией сферы развлечений и изменением модели потребления медиаконтента. В последние годы наблюдается экспоненциальный рост производства кинематографической продукции, сопровождающийся повышением требований пользователей к качеству поиска, персонализации контента и социальному взаимодействию вокруг просмотра фильмов. Современные потребители ожидают мгновенного доступа к информации о фильмах, возможности быстрого поиска по множественным критериям, интеграции с социальными сетями, персонализированных рекомендаций и удобных инструментов для оценки и обсуждения просмотренного контента. При этом многие организации, занимающиеся управлением медиаархивами, продолжают использовать устаревшие методы каталогизации, основанные на ручной обработке данных и локальных файловых системах, что приводит к потерям времени, снижению качества поиска и неэффективному использованию контента.

Разработка автоматизированной веб-системы управления кинотекой приобретает особую значимость в условиях, когда традиционные подходы к организации медиаархивов демонстрируют свою неэффективность. Ручная каталогизация фильмов, характерная для многих библиотек и архивов, не позволяет оперативно обрабатывать растущие объемы контента, обеспечивать качественные метаданные и предоставлять пользователям современные инструменты поиска и взаимодействия. Кроме того, отсутствие интеграции с внешними источниками данных о фильмах и современными веб-технологиями существенно ограничивает возможности таких систем по предоставлению актуальной и полной информации о кинопроизведениях.

**Цель данной курсовой работы** – разработать веб-систему управления кинотекой, которая позволит пользователям просматривать информацию о фильмах, оставлять отзывы, а администраторам – управлять контентом и пользователями системы.

## Задачи курсовой работы:
1. Провести анализ предметной области и определить функциональные требования к системе
2. Спроектировать архитектуру веб-приложения и структуру базы данных
3. Реализовать систему с использованием современных технологий веб-разработки
4. Обеспечить интеграцию с внешними API для получения информации о фильмах
5. Протестировать функциональность системы и проанализировать результаты

**Актуальность работы** обусловлена растущей потребностью в специализированных системах управления мультимедийным контентом, а также необходимостью изучения современных подходов к разработке веб-приложений с использованием объектно-реляционного отображения и паттерна MVC.

# 1 Анализ предметной области и постановка задачи

Система управления кинотекой должна обеспечивать работу с следующими основными сущностями:

## Основные сущности системы:

- **Фильмы (Movies)** – центральная сущность системы, содержащая информацию о кинопроизведениях: название, оригинальное название, описание, год выпуска, продолжительность, рейтинг, постер, связи с жанром и режиссером.

- **Пользователи (Users)** – лица, зарегистрированные в системе для просмотра фильмов и оставления отзывов. Включают персональную информацию и уровень доступа.

- **Отзывы (Reviews)** – оценки и комментарии пользователей к фильмам, связывающие пользователей с конкретными фильмами.

- **Режиссеры (Directors)** – информация о создателях фильмов, включая биографические данные и связь со странами.

- **Актеры (Actors)** – информация об исполнителях ролей в фильмах.

- **Роли (Roles)** – связующая сущность между фильмами и актерами, определяющая персонажей.

- **Жанры (Genres)** – классификация фильмов по тематическим категориям.

- **Страны (Countries)** – справочник стран для связи с режиссерами и актерами.

## Функциональные требования к системе:

- Регистрация и аутентификация пользователей
- Просмотр каталога фильмов с фильтрацией и поиском
- Добавление и редактирование отзывов к фильмам
- Административная панель для управления контентом
- Интеграция с TMDB API для получения постеров фильмов
- Система ролей пользователей (обычный пользователь/администратор)

# 2 Проектирование системы

## 2.1 Архитектура системы

Для реализации системы была выбрана архитектура **Model-View-Controller (MVC)**, которая обеспечивает разделение логики приложения на независимые компоненты:

| Компонент | Описание |
|-----------|----------|
| **Model (Модель)** | Представляет структуру данных и бизнес-логику приложения. Включает классы сущностей (Movie, User, Review и др.) и контекст базы данных. |
| **View (Представление)** | Отвечает за пользовательский интерфейс и отображение данных. Реализован с помощью Razor Pages для динамического формирования HTML. |
| **Controller (Контроллер)** | Обрабатывает пользовательские запросы, взаимодействует с моделью и возвращает соответствующие представления. |

## 2.2 Проектирование базы данных

База данных спроектирована с использованием принципов нормализации для обеспечения целостности данных и эффективности запросов.

### 3.3.2 Модель данных Movie

Основная модель фильма содержит:
- Обязательные поля: Title, OriginalTitle, ReleaseYear, Duration
- Рейтинг с типом decimal(3,1) для точности
- Навигационные свойства для связей с Genre, Director, Roles, Reviews
- Временные метки CreatedAt и UpdatedAt

```csharp
public class Movie
{
    public int MovieID { get; set; }
    
    [Required]
    [StringLength(255)]
    public string Title { get; set; }
    
    [Column(TypeName = "decimal(3,1)")]
    public decimal Rating { get; set; }
    
    // Навигационные свойства
    public int GenreID { get; set; }
    public Genre Genre { get; set; }
    
    public ICollection<Review> Reviews { get; set; } = new List<Review>();
}
```

# Приложение А. Структура базы данных

## Основные таблицы системы kursa4

```sql
-- Таблица фильмов
CREATE TABLE Movies (
    MovieID int NOT NULL AUTO_INCREMENT,
    Title varchar(255) NOT NULL,
    OriginalTitle varchar(255) NOT NULL,
    Description longtext,
    ReleaseYear int NOT NULL,
    Duration int NOT NULL,
    Rating decimal(3,1) NOT NULL,
    PosterURL longtext,
    GenreID int NOT NULL,
    DirectorID int NOT NULL,
    CreatedAt datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UpdatedAt datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (MovieID)
) CHARACTER SET=utf8mb4;

-- Таблица пользователей
CREATE TABLE Users (
    Id int NOT NULL AUTO_INCREMENT,
    FirstName longtext CHARACTER SET utf8mb4 NOT NULL,
    LastName longtext CHARACTER SET utf8mb4 NOT NULL,
    Email longtext CHARACTER SET utf8mb4 NOT NULL,
    Rank longtext CHARACTER SET utf8mb4 NOT NULL,
    RegistrationDate datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    Password longtext CHARACTER SET utf8mb4 NOT NULL,
    PRIMARY KEY (Id)
) CHARACTER SET=utf8mb4;

-- Таблица отзывов
CREATE TABLE Reviews (
    ReviewID int NOT NULL AUTO_INCREMENT,
    MovieID int NOT NULL,
    UserID int NOT NULL,
    Rating int NOT NULL,
    Text longtext CHARACTER SET utf8mb4 NOT NULL,
    PublicationDate datetime(6) NOT NULL,
    CreatedAt datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UpdatedAt datetime(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (ReviewID),
    CONSTRAINT FK_Reviews_Movies_MovieID 
        FOREIGN KEY (MovieID) REFERENCES Movies(MovieID) ON DELETE CASCADE,
    CONSTRAINT FK_Reviews_Users_UserID 
        FOREIGN KEY (UserID) REFERENCES Users(Id) ON DELETE CASCADE
) CHARACTER SET=utf8mb4;

CREATE TABLE Genres (
    GenreID int NOT NULL AUTO_INCREMENT,
    GenreName longtext CHARACTER SET utf8mb4 NULL,
    PRIMARY KEY (GenreID)
) CHARACTER SET=utf8mb4;

CREATE TABLE Directors (
    DirectorID int NOT NULL AUTO_INCREMENT,
    DirectorName longtext CHARACTER SET utf8mb4 NULL,
    Biography longtext CHARACTER SET utf8mb4 NULL,
    BirthDate datetime(6) NOT NULL,
    BirthYear int NULL,
    CountryID int NULL,
    PRIMARY KEY (DirectorID),
    CONSTRAINT FK_Directors_Countries_CountryID 
        FOREIGN KEY (CountryID) REFERENCES Countries(CountryID) ON DELETE RESTRICT
) CHARACTER SET=utf8mb4;

CREATE TABLE Actors (
    ActorID int NOT NULL AUTO_INCREMENT,
    ActorName longtext CHARACTER SET utf8mb4 NULL,
    Biography longtext CHARACTER SET utf8mb4 NULL,
    BirthDate datetime(6) NOT NULL,
    BirthYear int NULL,
    CountryID int NULL,
    PRIMARY KEY (ActorID),
    CONSTRAINT FK_Actors_Countries_CountryID 
        FOREIGN KEY (CountryID) REFERENCES Countries(CountryID) ON DELETE RESTRICT
) CHARACTER SET=utf8mb4;

CREATE TABLE Roles (
    RoleID int NOT NULL AUTO_INCREMENT,

    ActorID int NOT NULL,

    PRIMARY KEY (RoleID),
    CONSTRAINT FK_Roles_Movies_MovieID 
        FOREIGN KEY (MovieID) REFERENCES Movies(MovieID) ON DELETE CASCADE,
    CONSTRAINT FK_Roles_Actors_ActorID 
        FOREIGN KEY (ActorID) REFERENCES Actors(ActorID) ON DELETE CASCADE
) CHARACTER SET=utf8mb4;

CREATE TABLE Countries (
```
